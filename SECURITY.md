# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please **DO NOT** open a public issue.

Instead, please email security@supplychain.example.com with:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will respond within 48 hours and work with you to address the issue.

## Security Considerations

### Smart Contract Security

This contract has **NOT** been professionally audited. Use at your own risk.

Before deploying to mainnet:
1. Conduct professional security audit
2. Implement comprehensive testing
3. Use testnet extensively
4. Consider bug bounty program

### Known Limitations

- No upgrade mechanism (immutable after deployment)
- Owner has significant control (consider multisig)
- Platform fee changes affect existing orders
- No emergency pause mechanism

### Best Practices

When using this contract:
- Start with small amounts on testnet
- Verify all transactions before signing
- Use hardware wallets for large amounts
- Monitor contract events
- Keep private keys secure
